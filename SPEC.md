# Project Spec: HackerAgent Monorepo

## 1. Overview
HackerAgent is a "Browser-in-the-Loop" DevOps agent designed to be injected into the existing **ItrPlatform** web application. It functions as a bridge, allowing an AI agent to orchestrate tasks using the user's authenticated browser session.

The project is structured as a Monorepo using `pnpm` workspaces.

## 2. Tech Stack & Repository Structure

* **Monorepo Manager:** `pnpm`
* **Runtime:** Bun (for server packages), Node.js (for frontend build)

### Directory Structure
```text
/HackerAgent
├── pnpm-workspace.yaml
├── packages/
│   ├── frontend-sdk/      # Vite + React (Library Mode)
│   │   ├── src/
│   │   │   ├── components/ # Chat & Flow UI
│   │   │   ├── registry/   # Tool definitions
│   │   │   └── runtime/    # Execution engine
│   ├── agent/             # LangChain + Claude Logic
│   └── mcp-server/        # Bun WebSocket Server (MCP Host)
```

## 3. Package Specifications
### 3.1. Package: mcp-server
Role: The central communication hub and MCP Host.

Stack: Bun, ws (WebSocket), zod (Validation).

Responsibility:

Hosts the WebSocket Server at ws://localhost:4000.

Maintains connections to the Browser SDK.

Exposes the MCP Protocol endpoints (if integration with generic MCP clients is required later) or acts as the direct API host for the agent.

Router:

POST /chat: Receives user prompt from SDK -> Forwards to agent.

WS /connect: Establishes persistent connection to SDK.

Events (Server -> SDK):

EXECUTE_READ: Request immediate execution of a "read" function.

PROPOSE_FLOW: Request rendering of a "write" plan.

CHAT_STREAM: Stream AI text response to UI.

### 3.2. Package: agent
Role: The Brain.

Stack: LangChain (JS/TS), @langchain/anthropic, zod.

Logic:

Initializes the LangChain Agent with Claude.

Tool Definitions:

ReadTool:

Input: { functionId: string, arguments: object }

Behavior: Sends EXECUTE_READ to mcp-server -> Waits for Result -> Returns Observation.

PlanTool:

Input: { intent: string, steps: [{ functionId, arguments, title }] }

Behavior: Sends PROPOSE_FLOW to mcp-server -> Returns "Plan sent to user".

### 3.3. Package: frontend-sdk
Role: The UI and Execution Engine.

Stack: Vite (Library Mode), React, TailwindCSS.

Build Output: A single bundled sdk.js and style.css.

Key Modules:

A. The UI Layout (Split Pane)
The SDK injects a full-screen overlay or a substantial modal into the host app.

Left Pane (Chat Interface):

Standard chat UI (User bubbles, AI bubbles).

Streams text responses from agent.

Right Pane (Flow Visualizer):

Empty State: "Waiting for plan..."

Plan State: Renders the list of FlowNodes generated by the PlanTool.

Action Bar: "Run Flow", "Edit Parameters", "Cancel".

B. The Function Registry
Stores generic "Pure Function" definitions in localStorage.

Type Definition:

```ts
type FunctionDefinition = {
  id: string;
  name: string;
  description: string;
  code: string; // e.g. "return fetch('/api/v1/cluster/' + args.id)"
  parameters: { name: string; type: string }[];
};
```
C. The Execution Engine
Sandboxing: Uses new Function() to execute the code string.

Context: Passes window (for fetch/cookies) and args to the function.

Lifecycle:

Receives PROPOSE_FLOW.

User reviews and clicks "Run".

Iterates through nodes:

PENDING -> RUNNING -> SUCCESS / FAILED.

Updates UI status in real-time.

## 4. Data Models (Shared)
Flow Plan
```ts
type FlowPlan = {
  planId: string;
  intent: string;
  nodes: FlowNode[];
};

type FlowNode = {
  id: string;
  functionId: string;
  title: string;
  arguments: Record<string, any>;
  status: "pending" | "running" | "success" | "failed";
  result?: any;
};
```
## 5. Development Workflow
Start Server:

```bash
# Root
pnpm --filter mcp-server run dev  # Starts Bun WS server on 4000
```

Start Agent:

```bash
# Root
pnpm --filter agent run dev      # Links Agent logic to Server
```

Start SDK:

```bash
# Root
pnpm --filter frontend-sdk run dev # Vite dev server on 3000
```

Integration:

Open existing ItrPlatform (or a test HTML page).

Inject: <script type="module" src="http://localhost:3000/src/main.tsx"></script> (or the built SDK).

The HackerAgent UI appears.

## 6. Implementation Steps for AI

1. Scaffold Monorepo: Initialize pnpm workspace and folders.

2. Setup mcp-server: Create basic Bun WebSocket server with Zod schema validation for messages.

3. Setup agent: Configure LangChain with a dummy Claude setup and the two core Tools.

4. Setup frontend-sdk:

- Create the Split Pane Layout (Chat/Flow).

- Implement Registry (CRUD for functions).

- Implement Executor (The new Function runner).

5. Connect: Wire up the WebSocket PROPOSE_FLOW event to trigger the React state update in the Right Pane.
